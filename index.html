Проблема Решение Пример Стек

AI-микросервис для
автоматизации колл-центра

К нам обратился крупный сервис доставки и логистики (E-commerce, NDA), ежедневно обрабатывающий тысячи телефонных заказов. Из-за ошибок операторов при ручной фиксации данных часть доставок выполнялась некорректно или с задержками.

Какую проблему мы выявили

Оператор одновременно общался с клиентом, фиксировал адрес и детали заказа, а затем вручную переносил данные в CRM. В режиме реального времени это вызывало потерю фокуса и критичные ошибки при вводе.

Дополнительно отсутствовал прозрачный контроль качества звонков и единый формат хранения информации. Ключевой вывод: проблема не в отдельных сотрудниках, а в модели работы, где человек выполняет несколько несовместимых задач одновременно.

Решение

Мы разработали AI-микросервис, который обрабатывает записи звонков, извлекает ключевые данные и автоматически передаёт их в CRM в структурированном виде.

Двухрежимная обработка

Режим Address Only извлекает критические поля для заказа, а Full Transcript сохраняет полную расшифровку с ролями и таймкодами для контроля качества.

Асинхронный пайплайн

API сразу возвращает статус, задачи уходят в очередь, а воркеры обрабатывают аудио в фоне без блокировок под высокой нагрузкой.

Контроль стоимости и отказоустойчивость

Сервис считает стоимость по длительности и токенам, применяет retry-политику и остаётся устойчивым даже при деградации AI-провайдера.

Архитектура решения

Двухрежимная обработка

Address Only — для операционного ввода заказа, Full Transcript — для полной аналитики и контроля качества звонков.

Асинхронная обработка

Очереди и фоновые воркеры позволяют обрабатывать тысячи звонков без блокировок и масштабироваться горизонтально.

Финансовая логика и retry

Гибридный анализ аудио, расчёт затрат и экспоненциальные повторы защищают процесс от сбоев и перерасхода.

Мониторинг через Telegram

Telegram-бот показывает баланс, статус ключей и визуальную аналитику, чтобы управлять системой без доступа к инфраструктуре.

Запуск после звонка

После завершения разговора аудиозапись автоматически отправляется в AI-микросервис из телефонии.

Приём задачи API

API принимает задачу, валидирует вход и мгновенно возвращает статус обработки без ожидания результата.

Фоновая обработка

Воркеры извлекают адрес, детали заказа и формируют структурированный JSON по единому формату.

Автозапись в CRM

Готовые данные автоматически синхронизируются с CRM без участия оператора на этапе ввода.

Контроль качества

По необходимости сохраняется полный транскрипт для разбора диалогов, аудита и обучения команды.
